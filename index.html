Geodesic

Geodesic is a Sass toolkit for making large systems out of small parts. The parts in question include functional/atomic classes, object-oriented classes, and useful patterns and mixins to make writing new classes easier. It’s a living pattern language that influences and is influenced by design systems that Mule builds for clients.

Principles

Coherent > Consistent

Some people like functional classes, other people like objects. Geodesic uses both, because sometimes one or the other is a more appropriate way of getting something done. Trying to shoe-horn everything into a single conceptual model is a great way to make certain stuff really hard to do. Instead, Geodesic has a few different concepts around which rules are organized. Choose the one that will make your code easiest to use and understand.

Descriptive > Concise

The best kind of documentation is when a component's name tells you everything you need to know about it. I feel comfortable trading a few extra bytes in a class name for improved developer comprehension. Code that developers don’t understand gets duplicated in a way they do understand. This is the answer to the mystery of why your stylesheets keep getting bigger.

Discrete > Dependent

The more components rely on one another, the harder it is to make a single change to the system without unintended side effects. Using smaller, more focused classes lets trade more class names for better fine-grained control of elements. If you do it right, you'll have to change styles less often.

Spacing

Padding and margin classes are built on a simple vocabulary of properties, directions, and values.

The standard units are multiples of the $space-unit variable. The included list of multiples has quarter, half, one, double, and quad, but it's easy to change the scale depending on your project.

Here are some example classes and their output rules:

.mar-t-one     => margin-top: 20px;
.mar-r-half    => margin-right: 10px;
.pad-y-quarter => padding-top: 5px; padding-bottom: 5px;
.pad-double    => padding: 40px;

You can also use the gutter keyword to use the current grid's gutter as a spacing variable! More on grids »

@media screen and (min-width: 0px) {
  .mar-gutter {
    margin: 25px;
  }
}

@media screen and (min-width: 540px) {
  .mar-gutter {
    margin: 20px;
  }
}

Grid

Geodesic ships with an 8 column grid that breaks down to a 4 column grid on small screens. But those parameters crazy-easy to change just by changing the values in settings/_grids.scss. You can specify as many grids and sub-grids as you want, and since they have associated media queries it's easy to build complex repsonsive layouts!
